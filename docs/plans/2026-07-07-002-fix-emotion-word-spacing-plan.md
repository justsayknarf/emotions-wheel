# fix: Enforce minimum spacing between emotion word coordinates

## Summary

Fix the colliding coordinates in `src/data/emotions.ts` — starting with Angry and Afraid at the identical point — and add a spacing lint script that fails when any two words would render as overlapping text at a desktop reference viewport. The lint guards future edits, including saves from the admin editor.

## Problem Frame

Words render centered at their exact coordinates with no collision handling, so close data points print on top of each other. Measured at a 1280×800 reference render: 23 surface-surface pairs overlap (always co-visible — "Angry/Scared", "Worried/Nervous" merge into single glyph piles even in the idle field), and 286 deep-deep pairs overlap (co-visible after a reveal). Angry and Afraid share the exact coordinate (0.70, −0.65). Euclidean nearest-neighbor spacing of 0.05 units is only ~29px horizontally on desktop, while a typical label is 60–110px wide — labels need roughly four times more horizontal than vertical separation, which is why a naive min-distance rule is the wrong check.

## Requirements

**Data fix**

- R1. No two surface words overlap at the reference render metrics; Angry and Afraid get distinct coordinates.
- R2. No two words of any depth overlap at the reference render metrics.
- R3. Every moved word stays within 0.10 coordinate units of its original position, keeps its quadrant, and preserves its cluster's relative layout — the coordinate is a meaningful valence/arousal datum, not a free layout variable.
- R4. `src/data/emotions.ts` remains byte-compatible with the admin serializer format (`serializeEmotions`), so admin-editor round-trips stay clean.

**Lint**

- R5. A script reports every overlapping pair with labels, depths, and coordinates, and exits non-zero when violations exist.
- R6. The script runs via `pnpm lint:spacing` and is chained into `pnpm lint`.
- R7. The script fails loudly if it parses zero entries or fewer entries than the file contains, guarding against serializer format drift silently disabling the check.

## Key Technical Decisions

- **Overlap model: estimated label bounding boxes at a 1280×800 reference viewport, not euclidean distance.** Two words overlap when their horizontal gap is less than half the sum of their estimated label widths (plus 8px padding) *and* their vertical gap is under one line height (18px). Width estimate: label length × 7.7px for surface (14px font), × 6.6px for deep (12px font). This is the check that actually predicts the rendered collisions; euclidean distance over- and under-flags.
- **Desktop reference only.** At mobile widths no spacing fixes within circumplex semantics can prevent overlap (words would need ~0.36 units of horizontal separation); mobile readability is a separate rendering problem, out of scope here.
- **Parse `emotions.ts` with a regex keyed to the serializer's one-entry-per-line format.** The file header marks it auto-generated by `serializeEmotions`, which emits a stable single-line shape per entry — no TypeScript runtime needed for the lint. R7's count guard protects this assumption.
- **Surface words are repositioned by hand; deep words by a bounded relax script.** The 23 surface pairs are the always-visible anchors and deserve semantic judgment (e.g., Afraid moves toward its fear cluster, away from Angry). The 286 deep pairs are too many to hand-tune; an iterative pairwise-repulsion pass with a 0.10 displacement bound resolves most, and residuals are finished by hand.
- **Coordinates round to 2 decimals** to match the serializer's `toFixed(2)` output (minimum meaningful step: 0.01).

## Implementation Units

### U1. Spacing lint script

- **Goal:** `pnpm lint:spacing` reports overlap violations and gates `pnpm lint`.
- **Requirements:** R5, R6, R7
- **Dependencies:** none
- **Files:**
  - `scripts/lint-emotion-spacing.mjs` — new
  - `package.json` — add `lint:spacing`; chain into `lint`
- **Approach:** Plain Node script: read `src/data/emotions.ts`, regex-parse entries, apply the bounding-box overlap model from the KTDs, print violations grouped by pair depth (surface-surface first — they are the always-visible cases), exit 1 on any violation. Cross-check the parsed count against a raw count of `{ id:` occurrences and exit 1 on mismatch.
- **Patterns to follow:** `src/admin/lib/serialize.ts` defines the exact line format the regex must match.
- **Test scenarios:** the repo has no test runner; the script is verified by direct runs.
  - Run against current (unfixed) data: exits 1, reports 23 surface-surface pairs including Angry ↔ Afraid at distance 0.
  - Run against fixed data (after U2/U3): exits 0 with a summary line.
  - Corrupt the format assumption (e.g., point the script at a file with multiline entries): exits 1 with a parse-count error, not a false pass.
  - `pnpm lint` fails when a violation exists and passes when clean.
- **Verification:** script output names real overlapping pairs observed in the rendered field (Angry/Scared, Worried/Nervous, Lonely/Ashamed).

### U2. Reposition surface words

- **Goal:** Zero surface-surface violations; the idle field renders every surface word legibly.
- **Requirements:** R1, R3, R4
- **Dependencies:** U1
- **Files:** `src/data/emotions.ts`
- **Approach:** Hand-adjust the ~25 words involved in the 23 flagged pairs, keeping each within 0.10 of its original position and semantically placed: Afraid separates from Angry toward the fear cluster; the Frustrated/Irritated/Upset/Anxious knot in the activated-negative quadrant spreads along its arousal band. Edit in the serializer line format (or through the admin editor, which writes it).
- **Test scenarios:** `Test expectation: none — data-only change; U1's lint is the acceptance check.`
- **Verification:** `pnpm lint:spacing` reports zero surface-surface violations; idle field screenshot shows no merged labels; recorded diary entries are unaffected (entries store pin coordinates and word ids, never word positions — no migration).

### U3. Relax deep word positions

- **Goal:** Zero remaining violations of any depth.
- **Requirements:** R2, R3, R4
- **Dependencies:** U1, U2
- **Files:**
  - `scripts/relax-emotion-spacing.mjs` — new
  - `src/data/emotions.ts`
- **Approach:** Iterative pairwise repulsion over the overlap model: each iteration pushes overlapping pairs apart along their separation vector, surface words held fixed (U2 already placed them), displacement clamped to 0.10 from original and coordinates clamped inside (−0.98, 0.98), rounded to 2 decimals. Rewrite the file in serializer format. Run lint; finish stubborn residual pairs by hand. Keep the script in `scripts/` for reuse after future vocabulary edits.
- **Test scenarios:** the repo has no test runner; the relax pass is verified through the lint and the running app.
  - After the relax run, `pnpm lint:spacing` exits 0 (or lists only residuals that are then fixed by hand to reach 0).
  - Spot-check displacement bound: no word moved more than 0.10 from its pre-relax coordinate, no word changed quadrant.
  - In the app, dwell-reveal in the angry cluster (with the reveal cap from `docs/plans/2026-07-07-001-fix-cap-deep-reveal-plan.md` in place) shows non-overlapping words.
- **Verification:** zero lint violations; dwell and pin reveals in the three dense clusters (angry, sad, joyful) render every revealed word legibly.

## Scope Boundaries

### Deferred to Follow-Up Work

- Running the spacing check inside the admin-save Vite plugin (`src/plugins/admin-save.ts`) and surfacing warnings in the admin UI at save time. The lint script covers the guard; live admin feedback is a nicer follow-up.
- Mobile-width readability (fewer rendered words or runtime label collision handling) — a rendering-strategy problem, not a data problem.

### Outside this change

- Runtime label nudging (rendering a word offset from its coordinate). The recorded coordinate is the primary datum; the display must not lie about positions. Spacing is fixed in the data.

## Risks & Dependencies

- Region descriptions (`getRegionDescription`) derive from word positions, so text for a given pin location may shift slightly after repositioning. Acceptable: descriptions are computed at pin time, and stored entries carry their own snapshot.
- The relax pass may not fully converge in the densest cluster within the 0.10 bound. Fallback is manual placement of residuals; if that proves impossible, the padding constant (8px) is the tuning knob — an execution-time judgment, not a plan change.
